# CodexRadar 任务规则

任务 ID：`codex-radar`

- 默认周期：每 10 分钟访问 `https://codexradar.com/current.json`。
- 状态来源：只使用 CodexRadar JSON 接口，不抓取网页 DOM。
- 基本原则：先确认、再通知；同一事件不重复通知。

## 会提醒什么

- 预测预提醒：`prediction.level` 连续 2 次为高概率时发送 warning；同一天最多一次。
- 完整窗口：接口给出同一窗口的 `opened_at` 和 `closed_at`，连续 2 次确认后发送 critical。
- 无窗直接重置：没有开启中的窗口，但最新完成记录显示已经直接重置，连续 2 次确认后发送 critical。

## 怎么认定无窗直接重置

- 候选记录的开始和结束时间相同。
- 或窗口时长为 0。
- 或窗口文案是“无窗”/`No window`。

这类通知会用 `Codex 使用限制已直接重置` 作为标题，让它和普通速蹬窗口区分开。

## 数据优先级

- 完整 `current_window` 优先。
- 当前没有开启窗口时，才会用完整 `last_window`。
- 当前窗口仍开启时，不用旧 `last_window` 报警。

## 去重和抑制

- 正式通知按事件 ID 去重；没有 ID 时用时间、来源、标题等稳定字段生成去重键。
- 预测预提醒按服务时区的本地日期去重。
- 已知误报可以在代码内按事件 ID 或来源 URL 抑制。

## 不提醒什么

- 缺少开始或结束时间的窗口记录。
- 当前仍开启、还没关闭的窗口。
- 预测等级不是高概率的普通波动。
- 被抑制列表命中的已知误报。
